# repodump

A Python script that generates an LLM-friendly text file containing directory structure and file contents with comprehensive filtering options. Perfect for preparing repository content for analysis by language models.

This script was generated by an LLM using the `PROMPT.MD` file as input. Therefore, it may have bugs that will be fixed over time. For a more detailed description of the script and its intended behavior, please read the `PROMPT.md` file.

## Installation
No installation required! The script uses only Python standard library modules and works with Python 3.6+. Simply download `repodump.py` and run with Python.

## Usage

### Basic Usage

```bash
python repodump.py /path/to/repository
```
### Common Examples

**Generate only directory structure:**
```bash
python repodump.py /path/to/repo -t 
```

**Generate only file contents:**
```bash
python repodump.py /path/to/repo -c 
```

**Filter for Python and C files only:**
```bash
python repodump.py /path/to/repo -f *.py -f *.c
```

## Command Line Arguments

### Required Arguments

- `directory` - Path to the input repository or directory

### Optional Arguments

| Short | Long | Description |
|-------|------|-------------|
| `-o` | `--output` | Output file path (default: `repodump.txt`) |
| `-t` | `--tree` | Only include directory structure, not file contents |
| `-c` | `--contents` | Only include file contents, not directory structure |
| `-i` | `--ignore-gitignore` | Ignore `.gitignore` files |
| `-f` | `--filter` | Only include files matching these patterns |
| `-F` | `--filter-files` | Only include file contents for files matching these patterns |
| `-e` | `--exclude` | Exclude files matching these patterns |
| `-E` | `--exclude-files` | Exclude file contents for files matching these patterns |
| `-x` | `--exclude-from` | File containing patterns to exclude (gitignore format) |
| `-I` | `--include` | Include files matching these patterns (overrides exclusions) |
| `-p` | `--prompt` | Add prompt text at the bottom of the output |
| `-q` | `--quiet` | Don't output summary information |

Note: To add multiple patterns with the same argument (such as multiple exclude patterns), you must use the corresponding argument flag before each pattern. For instance, to exclude two patterns, you must use `python repodump.py /path/to/repo -e <pattern1> -e <pattern2>`.

## Pattern Matching

The script uses gitignore-style patterns for filtering:

- `*.py` - Match all Python files
- `src/` - Match the src directory
- `*.log` - Match all log files
- `/config.json` - Match config.json only in the root
- `**/*.test.js` - Match test files in any subdirectory

### Pattern Application Order

1. **Filter patterns** (`-f`): If specified, only files matching these patterns are considered
2. **Exclude patterns** (`-e`): Files matching these patterns are excluded (including `.gitignore` and always-excluded patterns like `.git/`)
3. **Include patterns** (`-I`): Files matching these patterns are re-included, overriding exclusions
4. **Content-specific filters** (`-F`, `-E`): Applied after structure filters to control file content inclusion
